# AI Workflow Documentation

## Tools Used

- **Claude (Anthropic)** — primary development assistant for scaffolding and initial logic.
- **Google Gemini** — used for advanced debugging, fixing visual tile orientation, and final alignment of the assets.

---

## Workflow Summary

The project was built using an AI-assisted iterative process. While the initial structure was generated by AI, significant manual intervention and directed debugging were required to align the visual assets with the logical grid.

### Step 1 — Project Scaffolding (Claude)
Shared the full task specification. Claude generated the backend API (Node/Express), the basic frontend structure (Vanilla JS), and the initial test suite.

### Step 2 — Asset Integration & Initial Logic
Uploaded `map.ascii` and the PNG assets. The AI attempted to write the `getPathTile` function to handle neighbor detection for the path tiles (`#`).

### Step 3 — Advanced Debugging & Refinement (User + Gemini)
This was the most critical phase where the AI-generated code failed to account for the real-world orientation of the PNG files.
- **The Issue:** Straight paths were rotated 90 degrees, and corner tiles were "pointing" in the wrong directions.
- **My Role:** I identified that the visual output didn't match the map logic. I implemented `console.log` debugging to track how the N/S/E/W neighbors were being detected.
- **The Fix:** I provided the AI with the specific log outputs and visual feedback. Together, we re-mapped the rotation values (`0`, `90`, `180`, `270`) to match the actual "base" orientation of the provided images.

### Step 4 — Final Polish & UX Enhancement (User + Gemini)
This stage focused on transforming a functional grid into a premium user interface.
- **Visual Clarity:** Implemented dynamic `zoom` and `responsive scaling` to ensure the map remains legible and comfortable to use on high-resolution monitors.
- **Interactive Tooltips:** Replaced static, hard-coded numbers on assets with animated CSS tooltips. Cabana numbers now appear elegantly on hover, maintaining a clean "luxury resort" aesthetic.
- **Legend & Asset Consistency:** Refined the map legend to use actual asset textures (water, cabana icons) and synchronized CSS filters for the "Booked" state to ensure perfect visual alignment with the live map.
- **UI Stability:** Integrated `user-select: none` and `draggable: false` properties across the grid. This prevents accidental blue highlighting or "ghost image" dragging, ensuring the map feels like a native application.

## Observations

- **AI Strength:** Excellent at generating boilerplate code, setting up testing environments (Jest/Playwright), and handling case-insensitive string matching.
- **Human Role:** The AI could not "see" the resort map. I had to act as the bridge between the logical "neighbor detection" and the visual "image rotation." Without manual testing and log analysis, the map would have remained visually broken despite the "correct" logic.
- **Testing:** Automated tests were used to ensure that the "logical" map stayed intact while we were "fixing" the visual layer.
